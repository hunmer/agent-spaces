# i18n 中英文多语言切换设计文档

**日期**: 2026-05-06
**状态**: 待实施
**范围**: packages/web 前端全量组件

## 1. 背景

Agent Spaces 前端当前所有用户可见文本均为硬编码英文。需要添加中英文多语言切换能力，提升中文用户体验。

## 2. 核心决策

| 决策 | 选择 | 理由 |
|------|------|------|
| i18n 库 | next-intl | 专为 Next.js 设计，App Router 原生支持，TypeScript 类型安全 |
| 运行模式 | Client-Only | 项目绝大部分组件是 `"use client"`，无需 Server Components 翻译 |
| 切换方式 | 手动切换 + localStorage | 即时生效无需刷新，不改路由结构 |
| 翻译文件 | 单文件 en.json / zh.json，按 namespace 分区 | 简单直接，避免碎片化 |
| 回退机制 | 缺失 key 回退英文 | 确保永不显示空白 |

## 3. 架构设计

### 3.1 文件结构

```
packages/web/src/
  locales/
    en.json          # 英文翻译（主文件）
    zh.json          # 中文翻译
    index.ts         # 加载逻辑 + 类型导出
  components/
    locale-provider.tsx  # LocaleProvider + useLocale
```

### 3.2 LocaleProvider

`LocaleProvider` 包裹 `NextIntlClientProvider`，提供 locale state 和切换函数。

```typescript
// src/components/locale-provider.tsx
'use client';

import { createContext, useContext, useState, useEffect, useCallback } from 'react';
import { NextIntlClientProvider } from 'next-intl';
import en from '@/locales/en.json';
import zh from '@/locales/zh.json';

const STORAGE_KEY = 'locale';
const DEFAULT_LOCALE = 'zh';

const messages = { en, zh } as const;
type Locale = keyof typeof messages;

interface LocaleContextType {
  locale: Locale;
  setLocale: (locale: Locale) => void;
}

const LocaleContext = createContext<LocaleContextType>(null!);

export function LocaleProvider({ children }: { children: React.ReactNode }) {
  const [locale, setLocaleState] = useState<Locale>(DEFAULT_LOCALE);

  useEffect(() => {
    const saved = localStorage.getItem(STORAGE_KEY) as Locale | null;
    if (saved && messages[saved]) {
      setLocaleState(saved);
    }
  }, []);

  const setLocale = useCallback((newLocale: Locale) => {
    setLocaleState(newLocale);
    localStorage.setItem(STORAGE_KEY, newLocale);
    document.documentElement.lang = newLocale === 'zh' ? 'zh-CN' : 'en';
  }, []);

  return (
    <LocaleContext.Provider value={{ locale, setLocale }}>
      <NextIntlClientProvider locale={locale} messages={messages[locale]}>
        {children}
      </NextIntlClientProvider>
    </LocaleContext.Provider>
  );
}

export function useLocale() {
  return useContext(LocaleContext);
}
```

### 3.3 layout.tsx 集成

```typescript
// 修改后的包裹结构
<ThemeProvider>
  <LocaleProvider>        {/* 在 ThemeProvider 内侧 */}
    <AuthGuard>
      <AppShell>{children}</AppShell>
    </AuthGuard>
  </LocaleProvider>
</ThemeProvider>
```

LocaleProvider 在 ThemeProvider 内侧，确保 locale state 在 AuthGuard 之前就绪。

### 3.4 翻译文件结构

按功能模块划分 namespace：

```jsonc
// en.json
{
  "common": { "save": "Save", "cancel": "Cancel", ... },
  "settings": { "title": "Settings", ... },
  "sidebar": { ... },
  "agent": { ... },
  "llm": { ... },
  "chat": { ... },
  "composer": { ... },
  "issue": { ... },
  "editor": { ... },
  "terminal": { ... },
  "git": { ... },
  "workspace": { ... },
  "home": { ... },
  "login": { ... },
  "projectSettings": { ... }
}
```

zh.json 的 key 结构与 en.json 完全一致。

### 3.5 组件改造模式

```typescript
// Before:
<span>Settings</span>

// After:
import { useTranslations } from 'next-intl';
const t = useTranslations('settings');
<span>{t('title')}</span>
```

- 公共文本（save/cancel/delete 等）用 `useTranslations('common')`
- 同一组件需要多个 namespace 时可多次调用
- 带变量的文本用 ICU 插值：`t('commentCount', { count: 5 })`

## 6. 安全与回退

- 缺失翻译 key 回退英文原文，不显示空白
- 开发模式下比较 en.json 和 zh.json 的 key 集合，缺失的 key 输出 console.warn
- 不修改 `next.config.ts`、路由结构、后端代码、Zustand stores
