# 安装部署

## 环境要求

在开始之前，确保你的系统已安装：

- **Node.js** >= 20
- **pnpm** >= 9
- **Git**

## 安装步骤

### 1. 克隆项目

```bash
git clone https://github.com/nicepkg/agent-spaces.git
cd agent-spaces
```

### 2. 安装依赖

```bash
pnpm install
```

### 3. 配置认证密钥

首次使用需要设置 Secret Key，用于登录认证。

创建认证配置文件：

```bash
mkdir -p ~/.agent-spaces-data
echo '{"secretKey": "your-secret-key"}' > ~/.agent-spaces-data/auth.json
```

将 `your-secret-key` 替换为你自己的密钥，这个密钥将用于登录时验证身份。

### 4. 配置 AI 模型（可选但推荐）

根据你想使用的 Agent 运行时，配置对应的 API Key：

**使用 Claude Code 运行时：**

```bash
export ANTHROPIC_API_KEY=sk-ant-xxx
```

**使用 OpenAI Codex 运行时：**

```bash
export CODEX_API_KEY=sk-xxx
```

也可以在后续的 Agent 预设配置界面中填写 API Key。

### 5. 启动服务

```bash
pnpm dev
```

启动后：

- **后端服务**：http://localhost:3100
- **前端页面**：http://localhost:3000

打开浏览器访问 `http://localhost:3000` ，使用你设置的 Secret Key 登录。

## 环境变量参考

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `3100` | 后端服务端口 |
| `HOST` | `0.0.0.0` | 后端监听地址 |
| `AGENT_SPACES_DATA_DIR` | `~/.agent-spaces-data` | 数据存储目录 |
| `ANTHROPIC_API_KEY` | - | Claude Code 运行时 API Key |
| `CODEX_API_KEY` | - | Codex 运行时 API Key |
| `NEXT_PUBLIC_WS_PORT` | `3100` | WebSocket 连接端口 |

## Docker 部署

如果需要容器化部署：

```bash
# 构建后端镜像
pnpm build:docker:server

# 构建前端镜像
pnpm build:docker:web
```

## 常见问题

### 端口被占用

修改 `PORT` 环境变量指定其他端口，同时设置 `NEXT_PUBLIC_WS_PORT` 为相同值。

### pnpm install 失败

确保 Node.js 版本 >= 20，pnpm 版本 >= 9。可以用 `node -v` 和 `pnpm -v` 检查版本。

### 登录失败

检查 `~/.agent-spaces-data/auth.json` 中的 `secretKey` 是否与登录时输入的一致。
