# Agent 系统

Agent 是 Agent Spaces 的核心执行单元。不同的 Agent 角色各司其职，协同完成开发任务。

## Agent 角色

系统定义了六种 Agent 角色：

### 调度者（Scheduler）

任务分发中心。接收新议题后，调度者判断需要哪些角色参与，并按顺序唤醒对应的 Agent。

### 策划者（Planner）

需求分析师。分析议题描述，制定执行计划，将需求拆解为具体的执行步骤和任务。

### 任务创建者（Task Creator）

计划执行者。将策划者的计划同步为系统中的任务，设定依赖关系和执行顺序。

### 执行者（Executor）

代码写手。接收任务后阅读代码、搜索上下文、修改文件、运行测试，完成实际的编码工作。

### 审核者（Reviewer）

代码审查员。检查执行者的代码变更，确保质量、风格和正确性。

### 提交者（Commit）

版本管理。分析代码变更，生成 conventional commit message，执行 Git 提交。

### Bot（可选）

外部集成角色。通过飞书或企业微信与外部用户交互，接收命令并返回执行结果。

## Agent 运行时

每个 Agent 基于特定的 AI 运行时运行：

| 运行时 | 说明 | 适用模型 |
|--------|------|----------|
| Claude Code | 基于 Anthropic Claude Agent SDK | Claude 系列模型 |
| OpenAI Codex | 基于 OpenAI Codex SDK | GPT 系列模型 |
| Open Agent SDK | 通用 Agent SDK | 多模型支持 |

## Agent 预设

Agent 预设（Preset）是 Agent 的配置模板。每个预设定义了：

- **角色** — Agent 担任的职责
- **运行时** — 使用的 AI 运行时
- **模型** — 具体的 AI 模型
- **API Key** — 模型访问凭证
- **系统提示词** — Agent 的行为指引
- **权限模式** — Agent 的操作权限
- **MCP 工具** — Agent 可使用的外部工具
- **技能** — Agent 掌握的特殊能力

你可以在项目设置面板中配置 Agent 预设。同一个 Agent 角色可以创建多个预设，使用不同的模型或配置。

## 自动化编排

当一个议题被创建时，系统按以下链路自动编排：

```
议题创建 → 调度者接收 → 策划者分析 → 任务创建者拆分
→ 执行者编码 → 审核者检查 → 提交者提交 → 议题关闭
```

每一步的状态变更都通过 WebSocket 实时推送到前端，同时触发通知中心事件。

你也可以通过频道聊天中 @mention Agent 跳过编排流程，直接触发特定 Agent 执行。
